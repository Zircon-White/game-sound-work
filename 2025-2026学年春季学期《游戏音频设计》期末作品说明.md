## 一、项目介绍与故事叙事设计

本游戏是一个以声音为唯一媒介的互动恐怖体验。玩家在完全黑暗的视觉环境下，仅通过耳边的声音信息来判断空间、危险和剧情走向。



### 1\.1 玩家如何操作（交互规则）

游戏退化了所有复杂的 UI 指令，只保留一个核心操作：长按屏幕/松开屏幕。

- **长按**：代表主角“前进与探索”。手指按住期间，主角会进行动作（如起身、推门）或持续行走，主线进度随之累计。但前进发出的声音会持续吸引废弃船舱深处的怪物。

- **松开**：代表主角“静止与屏息”。手指离开屏幕，主角的所有动作声和脚步声瞬间切断。这是躲避怪物追踪、让怪物失去目标并离去的唯一方式。

### 1\.2 故事体验流程（主角经历了什么）

主角的遭遇随着按压积累的时间轴线性推进：

- **苏醒阶段**：主角在黑暗、开阔的钢铁船舱中醒来。四周是凄冷的风声和空旷的滴水回响。长按屏幕时，耳边传来主角挣扎着从冷硬的铁板上爬起、衣服摩擦的动作声。

- **探索与阻碍**：主角开始踩着金属网格板在货舱走廊中摸索。行至中段，耳边传来沉重的铁门卡死和铰链摩擦的牙酸声。主角必须持续按住屏幕蓄力，伴随着几声粗重的喘息与撞击，铁门终于被生硬地推开。

- **环境切换**：跨过铁门后，脚下的清脆金属踏板声消失，突兀地变为粘稠、湿润、带有拉丝质感的泥泞路面。周围的滴水声也变得更加沉闷，空间感收缩。

- **未知的捕食者（动态触发）**：在探索过程中，黑暗中随时会传来非人类的沉重拖行脚步声和诡异的低频率粗重咆哮。声音在主角周围的3D声场中移动，随着主角探索而加速迫近。若执意前行，怪物将逼近耳畔；只有及时松手，才能听着怪物的声线擦身而过。

### 1\.3 游戏结局走向

本游戏设计了两个结局：

- **失败**：当玩家在怪物逼近时未及时收步，怪物最终行进至极限距离。游戏瞬间静音背景环境，爆发尖锐的Jump\-scare捕食声和血肉撕裂咀嚼音。视效泛起高饱和度血红，重载回大厅。

- **胜利**：若玩家成功克制恐惧，在红影拉锯中走完主线设定的路程，耳边将传来沉重舱门被大风吹开的轰鸣，海浪与海鸥的空旷声音涌入，压抑的低频底噪完全消退。屏幕缓缓褪温至纯白，平稳落幕。

## 二、音频系统架构与设计决策

本项目的开发不是一蹴而就的，分为四个迭代阶段：

1. 阶段一：Web Audio仿真合成，验证“一二三木头人”声音玩法

2. 阶段二：引入连续事件时间轴，解决静态脚步生硬重复问题

3. 阶段三：3D Panner与声音解耦，怪物叫声、脚步实现空间动态漂移

4. 阶段四：增加滑行惯性与免死机制，增加恐怖反馈

### 2\.1 验证交互可行性

项目最初的灵感源于对“盲人视觉”恐怖游戏的探索。为了快速自证，开发初期使用Web Audio API的`OscillatorNode`和由数学公式计算出来的白噪声/粉红噪声缓冲口，由代码在浏览器里程序化合成了低频嗡鸣、滴水和敲击声，验证了交互可行性。



### 2\.2 时间轴重构

在阶段一中，只要按着按钮，代码就会像发报机一样反复触发单声脚步。当考虑加入“爬起、推门、泥地”等多变体验时，离散触发导致声音支离破碎。

优化方案：

- 引入`storyline`时间轴数组；

- 脚下的地面被定义为全局变量，时间推进至对应节点时直接改写地面标志位；

- 引入独立的主角动作音轨通道`actionGain`，实现事件动作与探索脚步的排他性播放。

### 2\.3 基于HRTF的3D声场与怪物声响解耦

在此阶段，发现原本怪物的脚步和叫声被绑定在同一个声轨中，无法表现“怪物边走边叫”的层次感，且怪物移动方向死板（仅从左声道出现、退回）。

优化方案：

将怪物声音解耦为`monster_step`与`monster_voice`两个独立实体，并接入三维空间音轨；设计圆形轨线算法，怪物退去时随机生成大偏转夹角，沿圆弧在玩家耳边漂移绕行，大幅提升声场包围感。



### 2\.4 加入惯性与免死

阶段三机制缺陷：听到怪物靠近后松手，怪物会瞬间回弹，戏剧张力不足。

阶段四优化：新增惯性与免死机制。松开按键后，怪物会依靠原有惯性继续贴近一段时间；惯性窗口期内，即使怪物进入致死距离，系统启动免死逻辑，将怪物位置锁定在安全临界值，营造“怪物贴耳却不会死亡”的紧张感。



## 三、音频设计模块的开发过程与触发逻辑

### 3\.1 玩家行动音效模块

#### 开发思路与痛点解决

初始版本脚步声使用间隔计时器重复播放单次脚步音频文件；调试时发现，调整播放速度后，计时器与音频时长无法对齐，极易出现声音卡顿、重叠爆音。



#### 实现方式

1. 将`player_metal`金属脚步、`player_mud`泥地脚步制作为带环境反射的循环音轨（`loop=true`）；

2. 按下触发`pressAction()`：淡入动作通道Gain节点，播放对应循环音轨；

3. 松开触发`releaseAction()`：不直接执行`Audio.pause()`（避免截断爆音），使用`setTargetAtTime(0, ctx.currentTime, 0.035)`平滑淡出音量至0。

### 3\.2 剧情时间轴模块

#### 开发思路

剧情触发起身、推门等关键动作时，若玩家持续长按屏幕，脚步声会与叙事动作音重叠，破坏沉浸体验。



#### 触发逻辑

搭建动作事件队列`eventQueue`与播放锁机制：

1. 主线时间轴推进至剧情阈值，触发动作事件；

2. 事件加入队列，强制暂停行走音效总线`pauseWalking()`；

3. 独占音频播放器，播放对应剧情音频（如`action_door.mp3`）；

4. 监听音频`ended`结束事件，清空播放锁；若玩家仍按住屏幕，平滑恢复脚步循环`resumeWalking()`。

### 3\.3 怪物AI追踪与惯性逼近模块

#### 基础机制

`monster_step.mp3`拖曳脚步、`monster_voice.mp3`嘶吼声两套音频独立运算：

- 玩家长按 → 怪物距离持续缩小（逼近加速）

- 玩家松开 → 惯性倒计时阶段怪物继续靠近（免死保护）→ 倒计时结束怪物后退，并生成随机逃逸角度

#### 1\) 随机方位切入与退散漂移

- **切入逻辑**：怪物刷新时，在平面极坐标系生成随机初始角度；

- **惯性免死**：松手后开启`monsterCoastDuration`惯性时间窗，怪物保持前冲；若距离进入致死区间，强制锁死在安全临界位置，弱化操作容错压力；

- **弧形退散**：危机解除后，怪物目标偏转区间设为`当前角度±[120°,240°]`，角度通过插值平滑过渡，实现绕行音效。

```JavaScript
let deltaAngle = monster.targetAngle - monster.angle;
deltaAngle = Math.atan2(Math.sin(deltaAngle), Math.cos(deltaAngle));
monster.angle += deltaAngle * dt * 0.95; // 弧形退散物理模拟
```



#### 2\) 基于位置的动态音频滤波器衰减

怪物极坐标实时转换为三维空间坐标：

$X = \sin(\theta) \times radius$

$Z = -\cos(\theta) \times radius$

坐标每320ms更新至两套音频挂载的`PannerNode`；同步控制低通滤波器截止频率：远处240Hz（低频沉闷、隔墙感）线性过渡至近处7600Hz（细节清晰、刺耳尖锐）。



#### 3\.4 状态结算模块

初代版本缺陷：死亡音效触发后底层计时未阻断，场景重载导致音效中途切断。

优化结算逻辑：

- **逃脱结局**：添加音频冲突保护窗口，搭配CSS渐变，画面6秒内缓慢淡化为纯白；

- **死亡结局**：锁定全部用户输入，静音环境底噪`bg_ambient`，保证惊吓音效完整播放。

## 四、关键音频设计模块开发与触发逻辑

### 4\.1 动作事件优先于行动脚步

时间轴触发起身、推门剧情时，行走音会与叙事音效叠加，破坏叙事氛围，为此设计**事件排他性触发逻辑**。

```JavaScript
function enqueueEvent(key) {
  if (eventQueue.includes(key) || currentEventKey === key) return;
  eventQueue.push(key);
  if (holding && !currentEventKey) {
    startNextEvent(); // 阻塞脚步声的播放
  }
}

async function startNextEvent() {
  if (!holding || currentEventKey || eventQueue.length === 0) return;
  pauseWalking(); // 暂时关停主角前进循环
  currentEventKey = eventQueue.shift();
  await playSingleton(currentEventKey, { restart: true, loop: false });
}
```

动作音频播放完成后触发`ended`回调，执行`finishCurrentEvent()`；若玩家仍长按屏幕，平滑恢复当前地形脚步循环，保证音效干净无冲突。



### 4\.2 怪物的方向漂移

为避免怪物原路折返的单调声场，退散阶段增加弧度角度补偿：

- **逼近状态**：锁定怪物当前角度，仅径向距离持续缩小；

- **松手退散**：随机左右偏转120°\~270°（1\.8\~3\.3弧度）生成目标偏转角度，沿圆弧绕行远离。

```JavaScript
if (monster.wasApproaching) {
  // 随机向左或向右偏转 120° 到 270°（1.8 到 3.3 弧度）
  const angleDeviation = (Math.random() >= 0.5 ? 1 : -1) * (1.8 + Math.random() * 1.5);
  monster.targetAngle = monster.angle + angleDeviation;
  monster.wasApproaching = false;
}
monster.distance += config.game.monsterRetreatSpeed * dt;
// 沿短弧插值偏转怪物角度
let deltaAngle = monster.targetAngle - monster.angle;
deltaAngle = Math.atan2(Math.sin(deltaAngle), Math.cos(deltaAngle));
monster.angle += deltaAngle * dt * 0.95;
```



### 4\.3 叫声与脚步独立随机间隔控制

怪物嘶吼不与脚步强制绑定，采用独立概率触发逻辑：

- 怪物距离较远：嘶吼触发概率低；

- 怪物逼近玩家：音量提升、低通滤波截止频率提高，同时嘶吼触发概率从远场值平滑拉升至近场最大值（`monsterVoiceChanceWhenFar` → `monsterVoiceChanceWhenNear`）。
